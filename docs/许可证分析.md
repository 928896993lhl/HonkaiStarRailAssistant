## 二、兼容性分析

### 2.1 AGPL-3.0 vs GPL-3.0（图片资源冲突）⚠️

**问题**：我们从 StarRailAssistant（GPL-3.0）直接复制了 26 张 PNG 模板图片。

GPL-3.0 的条款很清楚：衍生作品必须以相同许可证发布。图片一旦融入我们的项目，整个项目就受到 GPL-3.0 的约束。

**好消息**：AGPL-3.0 是 GPL-3.0 的**超集**。GPL-3.0 第 13 条明确写了：

> "Notwithstanding any other provision of this License, you have permission to link or combine any covered work with a work licensed under version 3 of the GNU Affero General Public License into a single combined work..."

翻译：**AGPL-3.0 和 GPL-3.0 是互相兼容的**。我们的 AGPL-3.0 项目包含 GPL-3.0 的图片是合法的。

**但有一个义务**：我们需要在文档中明确标注那些图片来自 StarRailAssistant (GPL-3.0)，就像我们在 commit message 中做的那样。

### 2.2 AGPL-3.0 vs LGPL-3.0（框架依赖）✅

MaaFramework 是 LGPL-3.0。LGPL 的设计目的就是允许非自由软件链接使用。

我们的项目**不包含** MaaFramework 的源代码，只在运行时链接（deps/bin 中的 .dll）。用户需要自行下载 MaaFramework 的 Release 包。

这在 LGPL-3.0 的「Combined Works」条款下完全合规。

### 2.3 MIT（MaaStarRail）✅

MIT 是最宽松的许可证，只要求保留版权声明。我们只是参考了 Pipeline JSON 的结构思路（非逐字复制），完全没有问题。

## 四、风险评估

### ❌ 不涉及侵权的理由

| 风险点 | 结论 | 依据 |
|--------|------|------|
| 使用了 GPL-3.0 图片 | 合规 | AGPL-3.0 与 GPL-3.0 兼容（GPL §13） |
| 使用了 LGPL-3.0 框架 | 合规 | LGPL 允许动态链接，我们不分发框架源码 |
| 参考了 MIT 项目结构 | 合规 | MIT 几乎没有限制 |
| 是否侵犯米哈游版权 | 低风险 | 图片是游戏 UI 截图，属于合理引用的功能性片段；项目不修改游戏文件，不破解协议 |

### ⚠️ 建议改进

| 优先度 | 建议 | 理由 |
|--------|------|------|
| 🔴 高 | **把 LICENSE 改为 GPL-3.0** | 与 SRA 的图片许可一致，避免 AGPL 的网络条款不必要地增加合规复杂度 |
| 🟡 中 | 在 README 加一句致谢声明 | 列出所有依赖和图片来源，明确各自的许可证 |
| 🟢 低 | 添加 NOTICE 文件 | 记录所有第三方资源的版权声明 |

## 五、推荐行动

### 5.1 立即更换 LICENSE 为 GPL-3.0

原因：
1. MaaFramework 是 LGPL-3.0（可被 GPL 包含）
2. SRA 图片是 GPL-3.0
3. AGPL 的「网络交互也触发开源」条款对桌面 GUI 工具没意义
4. GPL-3.0 对所有依赖都合规，没有多余负担

### 5.2 在 README 添加致谢段落

```markdown
## 📄 许可证与致谢

本项目基于 GNU General Public License v3.0 开源。

- 模板图片部分来自 [StarRailAssistant](https://github.com/Shasnow/StarRailAssistant) (GPL-3.0)
- Pipeline 结构参考 [MaaStarRail](https://github.com/VincenttHo/MaaStarRail) (MIT)
- 功能设计参考 [March7thAssistant](https://github.com/moesnow/March7thAssistant) (GPL-3.0)
- 由 [MaaFramework](https://github.com/MaaXYZ/MaaFramework) (LGPL-3.0) 强力驱动
```

### 5.3 不需要做的事

- ❌ 不需要删除 SRA 的图片（合规）
- ❌ 不需要联系原作者（开源许可证已授权）
- ❌ 不需要闭源（也没必要）

## 六、总结

**一句话：完全合规，唯一建议是把 LICENSE 从 AGPL-3.0 换成 GPL-3.0。**

所有依赖和引用的许可证互相兼容，不存在侵权风险。米哈游方面的风险属于所有游戏辅助工具的共性风险（不是许可证问题），而且我们不做内存修改/协议破解，纯图像识别模拟操作，相对安全。
